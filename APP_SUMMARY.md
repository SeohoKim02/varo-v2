# Varo V2 제품 개요

## 한 줄 설명

Varo V2는 악성재고와 점포별 부족재고를 연결해 이동 경로·수량·비용을 검토하고, 실행 전후 재고 상태를 확인하는 운영 의사결정 시스템이다.

## 해결하려는 문제

점포별 재고와 수요가 분리되어 있을 때 발생하는 과잉 보유, 품절 위험과 폐기 비용을 함께 분석하고, 실제 이동 가능 재고와 도착 수요 한도 안에서 실행할 수 있는 경로를 보여준다.

## 핵심 기능

1. 엑셀 업로드와 기본·DQN 샘플 로드
2. 악성재고 판단과 점포 간 이동 후보 생성
3. VHS 우선순위와 Greedy·DQN·Pareto 비교
4. DIRECT/VIA_DC 경로 및 차량 이동 시뮬레이션
5. 실제 재고 이동 전후 상태와 상위 3개 순차 적용
6. 민감도, 최적성 Gap과 전체 샘플 품질 검증
7. CSV·Excel·JSON 운영용 내보내기

## 알고리즘 구조

VHS가 현재 추천 우선순위를 구성하고 Greedy와 Pareto가 비교 기준을 제공한다. DQN은 자동 실행하지 않으며 사용자가 실행한 결과가 현재 데이터와 일치하고 품질 조건을 통과했을 때만 제한적으로 참고한다. 재고 전이 엔진은 이 순위를 변경하지 않고 실행 가능 수량만 복사본에서 검증한다.

## 재고 상태와 이동 원칙

현재 재고와 실제 수요 기준을 비교해 초과재고, 적정재고, 부족재고, 데이터 부족으로 구분한다. 출고 후 음수 재고를 막고, 이동 가능 재고와 도착 수요·부족량을 넘지 않으며, DIRECT와 VIA_DC 모두 이동 전후 상품 총량을 보존한다. 입력에 없는 수량이나 용량은 생성하지 않는다.

## 운영 화면

홈은 현재 데이터 정보, 전체 예상 절감액, 추천 후보 수와 재고 이동 시뮬레이션만 유지한다. 점포 카드는 상품·재고·수요·상태와 전후 변화를 표시하고, DC 카드는 경유 여부와 실제 처리 수량을 표시한다. 상위 3개 모드에서는 추천을 순서대로 적용해 중복 출고를 차단한다.

## 품질 검증

전체 샘플 품질 검증은 추천 생성, DQN 데이터 편향, Pareto 상태, 상세 민감도, 제약조건, 최적성 Gap과 오류·제외 내역을 비교한다. 빠른·표준·전체 범위는 사용자가 직접 실행하며 앱 시작 시 자동 분석하지 않는다.

## 현재 제공 범위

5개 운영 화면, 10점포·2DC 샘플, DQN 원본·균형형 비교, 재고 상태 전이, 차량 경로, 일반 내보내기와 자동 테스트가 연결되어 있다. 자세한 실행·데이터·배포 방법은 `README_V2.md`와 `DEPLOY_CHECKLIST.md`에서 확인한다.
