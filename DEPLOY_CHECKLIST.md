# Varo V2 배포·운영 확인 체크리스트

## 1. 실행

```powershell
cd C:\Users\user\OneDrive\Desktop\Projects\Varo\varo_v2
py -m streamlit run app_v2.py --server.headless true --server.port 8539
```

- [ ] `http://localhost:8539`가 HTTP 200을 반환한다.
- [ ] `http://localhost:8539/_stcore/health`가 HTTP 200을 반환한다.
- [ ] 일반 실행 주소 `http://localhost:8501`도 필요할 때 확인한다.
- [ ] 터미널에 warning, traceback, import error가 없다.

## 2. 저장소와 Cloud 준비

- [ ] `.gitignore`가 로컬 산출물에 적용된다.
- [ ] `.streamlit/secrets.toml`은 Git에 포함하지 않는다.
- [ ] 기본 배포는 `requirements.txt`를 사용한다.
- [ ] `requirements-dqn.txt`는 루트 requirements를 참조한다.
- [ ] Python 3.11을 선택해 로컬 검증 환경과 맞춘다.
- [ ] `outputs/dqn/`, `outputs/dqn_balanced_samples/`, `outputs/qa/`, 모델, 로그와 캐시는 배포에서 제외한다.

## 3. 홈 화면

- [ ] 현재 데이터명, 점포/DC/상품/추천 후보 수와 전체 예상 절감액이 표시된다.
- [ ] 홈에 추천 Top5 표, 상세 알고리즘 표, 다운로드 영역이 없다.
- [ ] 경로 1~3순위와 단일 경로/상위 3개를 선택할 수 있다.
- [ ] 이동 전, 이동 후, 전후 비교를 전환할 수 있다.
- [ ] 시뮬레이션 실행, 다시 실행, 속도와 전체 경로 표시가 동작한다.
- [ ] 점포명, 상품명, 재고 수량과 상태 배지가 잘리지 않는다.
- [ ] 재고 수준 막대와 출고·입고 수량이 표시된다.
- [ ] 상태 단계 바와 최대 4개 결과 KPI가 표시된다.
- [ ] 밝은 배경을 유지하고 검은 카드나 위젯이 없다.

## 4. 샘플 01

- [ ] DQN 샘플 01을 불러오면 2점포·1DC가 표시된다.
- [ ] 대표 DIRECT 또는 VIA_DC 경로의 이동 전후 재고를 확인한다.
- [ ] 출고량과 입고량이 같고 전체 재고가 보존된다.
- [ ] 차량과 점포·DC 이름이 겹치지 않는다.

## 5. 샘플 10

- [ ] DQN 샘플 10을 불러오면 10점포·2DC가 표시된다.
- [ ] DC01/DC02 이름과 ID가 구분된다.
- [ ] 선택 경로가 경유하는 DC만 강조된다.
- [ ] 상위 3개가 추천 순서대로 임시 재고에 적용된다.
- [ ] 앞 경로에서 사용한 재고를 다음 경로가 중복 출고하지 않는다.
- [ ] 실행할 수 없는 경로는 이유와 함께 건너뛴다.

## 6. 5개 화면

- [ ] 홈, 추천 실행, 경로 상세, 분석 및 검증, 데이터 관리가 모두 열린다.
- [ ] 페이지 이동 후 현재 데이터와 선택 경로가 유지된다.
- [ ] 추천 실행 표에 순위·상품·출발·도착·경로·수량·절감액·등급이 표시된다.
- [ ] 경로 상세에서 DIRECT는 한 구간, VIA_DC는 선택된 DC를 포함한 단계로 표시된다.
- [ ] 분석 및 검증에서 VHS, Greedy, DQN, Pareto, 민감도와 최적성 Gap을 확인할 수 있다.
- [ ] DQN 학습과 전체 샘플 품질 검증은 버튼을 누르기 전 실행되지 않는다.
